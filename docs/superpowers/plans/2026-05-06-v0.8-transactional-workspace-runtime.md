# v0.8 Transactional Workspace Runtime Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a pre-mutation gate (transactions) that stages `edit`-driven file changes, runs validators against a hermetic staged view, and requires explicit approval before any change lands in the real workspace, while integrating cleanly with shipped Phase 4 headless runtime and auto-compact.

**Architecture:** A new `src/workspace/transactions/` module owns the persistent tx object lifecycle (`$CLIQ_HOME/tx/<txId>/`). A `WorkspaceWriter` abstraction in `src/runtime/` lets `edit` write to either the real workspace (passthrough) or an overlay (overlay impl) without changing tool semantics. Apply uses a 3-stage protocol (preflight A → write B → record C) with deterministic record ids; abort uses a parallel AB0..AB9 protocol. Both share an under-lock crash recovery scanner. Tx events extend `HeadlessRuntimeEventType`; new session record kinds (`tx-applied`, `tx-aborted`) plug into existing replay and auto-compact range selection.

**Tech Stack:** TypeScript, Node.js `node:test`, ESM, no new runtime dependencies. `clonefile` (macOS) and `FICLONE` (Linux) reflinks for staged-view; symlink for bind paths; per-tx file locks layered on top of the existing session lock primitives.

**Spec reference:** `docs/superpowers/specs/2026-05-02-cliq-transactional-workspace-runtime-design.md`. When this plan says "see Section X", consult that spec.

Data flow at apply time:

```text
CLI (cliq tx apply <id>)
  → coordinator.apply(txId)
    → store.acquireTxLock(txId)
    Stage A: A1a (state + abort-progress + apply-progress checks) → A2..A5
    A6 release lock
    Stage B: B1 reacquire → B1a defense → B3 file writes per plan
    B5 release lock
    Stage C: acquire session > tx-store
      C4 scan-or-append txrec_apply_<txId> + clear activeTxId (1 fsync)
      C5..C7 transitions
    C8 schedule overlay cleanup
  → emit tx-applied runtime event
  → headless event factory wraps it; cliq run --jsonl streams it
```

```ts
  return `${OPEN_RECORD_ID_PREFIX}${txId}`;
}

  return `${APPLY_RECORD_ID_PREFIX}${txId}`;
}

export function abortRecordId(txId: string): string {
  return `${ABORT_RECORD_ID_PREFIX}${txId}`;
}

export function validatorSummaryFromTx(tx: Transaction): {
  blocking: { pass: number; fail: number };
  advisory: { pass: number; fail: number; names: string[] };
} {
  const summary = { blocking: { pass: 0, fail: 0 }, advisory: { pass: 0, fail: 0, names: [] as string[] } };
  for (const v of tx.validators ?? []) {
    if (v.severity === 'blocking') {
      if (v.status === 'pass') summary.blocking.pass++;
      else summary.blocking.fail++;
    } else {
      if (v.status === 'pass') summary.advisory.pass++;
      else { summary.advisory.fail++; summary.advisory.names.push(v.name); }
    }
  }
  return summary;
}
```

- [ ] **Step 3: Extend `SessionRecord` union**

Edit `src/session/types.ts`. Add three new variants at the end of the existing `SessionRecord` union. The `tx-opened` variant lets auto-compact pair an explicit `cliq tx open` with its matching terminal record (apply/abort) by `meta.txId`, so a multi-turn explicit transaction is treated as one non-splittable unit by the compact range selector.

```ts
export type SessionRecord =
  // ... existing variants unchanged
  | {
      id: string;          // deterministic: txrec_open_<txId>
      ts: string;
      kind: 'tx-opened';
      role: 'user';
      content: string;     // e.g. "Transaction tx_01HX... opened (explicit)"
      meta: {
        txId: string;
        txKind: 'edit';
        name?: string;     // user-provided label from `cliq tx open <name>`
        explicit: true;    // implicit per-turn tx do not write tx-opened records
      };
    }
  | {
      id: string;          // deterministic: txrec_apply_<txId>
      ts: string;
      kind: 'tx-applied';
      role: 'user';
      content: string;
      meta: {
        txId: string;
        txKind: 'edit';
        diffSummary: import('../workspace/transactions/types.js').DiffSummary;
        files: { creates: string[]; modifies: string[]; deletes: string[] };
        validators: {
          blocking: { pass: number; fail: number };
          advisory: { pass: number; fail: number; names: string[] };
        };
        overrides: import('../workspace/transactions/types.js').OverrideEntry[];
        artifactRef: string;
        ghostSnapshotId?: string;
      };
    }
  | {
      id: string;          // deterministic: txrec_abort_<txId>
      ts: string;
      kind: 'tx-aborted';
      role: 'user';
      content: string;
      meta: {
        txId: string;
        txKind: 'edit';
        reason: import('../workspace/transactions/types.js').AbortReason;
        failedValidators?: string[];
        files: { wouldHaveCreated: string[]; wouldHaveModified: string[]; wouldHaveDeleted: string[] };
        artifactRef: string;
        appliedPartial?: {
          partialFiles: string[];
          ghostSnapshotId: string;
          restoreConfirmed: boolean;
        };
      };
    };
```

Implicit per-turn transactions do **not** write `tx-opened` records — they are contained within a single turn and the existing turn-boundary rule already prevents splitting them. Only explicit `cliq tx open <name>` invocations write `tx-opened`.

```ts
    await fh.close();
```

```ts
// src/runtime/workspace-writer.ts
import { promises as fs } from 'node:fs';
import path from 'node:path';

export type WorkspaceWriter = {
  read(workspaceRelativePath: string): Promise<string>;
  replaceText(workspaceRelativePath: string, oldText: string, newText: string): Promise<void>;
};

export function createPassthroughWriter(cwd: string): WorkspaceWriter {
  return {
    async read(rel) {
      return fs.readFile(path.join(cwd, rel), 'utf8');
    },
    async replaceText(rel, oldText, newText) {

      await fs.writeFile(target, current.replace(oldText, newText), 'utf8');
    }
  };
```

```ts
// src/workspace/transactions/diff.ts
import { promises as fs } from 'node:fs';
import path from 'node:path';
import type { Diff, DiffEntry } from './types.js';

export async function computeDiff(cwd: string, overlayRoot: string): Promise<Diff> {
  const files: DiffEntry[] = [];
  await walk(overlayRoot, '', async (rel) => {
    const newContent = await fs.readFile(path.join(overlayRoot, rel), 'utf8');
    const oldContent = await fs.readFile(path.join(cwd, rel), 'utf8');
    files.push({ path: rel, op: 'modify', oldContent, newContent });
  });
  return { files, outOfBand: [] };
}

async function walk(root: string, prefix: string, visit: (rel: string) => Promise<void>): Promise<void> {
  let entries;
  try {
    entries = await fs.readdir(path.join(root, prefix), { withFileTypes: true });
  } catch (err) {
    if ((err as NodeJS.ErrnoException).code === 'ENOENT') return;
    throw err;
  }
  for (const entry of entries) {
    const next = prefix ? path.join(prefix, entry.name) : entry.name;
    if (entry.isDirectory()) {
      await walk(root, next, visit);
    } else if (entry.isFile()) {
      await visit(next);
    }
  }
}
```

```ts
export function summarizeDiff(diff: Diff): DiffSummary {
  const summary: DiffSummary = {
    filesChanged: diff.files.length,
    additions: 0,
    deletions: 0,
    creates: [],
    modifies: [],
    deletes: []
  };
  for (const f of diff.files) {
    if (f.op === 'modify') {
      summary.modifies.push(f.path);
      const oldLines = f.oldContent.split('\n');
      const newLines = f.newContent.split('\n');
      summary.additions += Math.max(0, newLines.length - oldLines.length);
      summary.deletions += Math.max(0, oldLines.length - newLines.length);
    }
  }
  return summary;
}
```

```ts
import { promises as fs } from 'node:fs';
import path from 'node:path';
import { execFile } from 'node:child_process';
import { promisify } from 'node:util';
import type { TxCopyMode } from '../config.js';

const execFileAsync = promisify(execFile);

export type CopyResult = 'reflinked' | 'copied';

export type StagedViewOptions = {
  cwd: string;
  overlayRoot: string;
  target: string;
  bindPaths: string[];
  copyMode: TxCopyMode;
  onWarn?: (message: string) => void;
};

export async function materializeStagedView(opts: StagedViewOptions): Promise<{ usedCopyFallback: boolean }> {
  await fs.mkdir(opts.target, { recursive: true });
  const bindSet = new Set(opts.bindPaths);
  let usedCopyFallback = false;
  await walkAndMaterialize(opts.cwd, opts.target, '', bindSet, async (src, dst) => {
    const result = await materializeFile(src, dst, opts.copyMode);
    if (result === 'copied' && opts.copyMode === 'auto') usedCopyFallback = true;
    return result;
  });
  // Apply overlay shadows
  await walkOverlay(opts.overlayRoot, opts.target);
  if (usedCopyFallback && opts.onWarn) {
    opts.onWarn('staged-view falling back to byte copy; reflink unsupported on this filesystem');
  }
  return { usedCopyFallback };
}

async function walkAndMaterialize(
  cwd: string,
  target: string,
  prefix: string,
  bindSet: Set<string>,
  copyOne: (src: string, dst: string) => Promise<CopyResult>
): Promise<void> {
  const fullSrc = path.join(cwd, prefix);
  const entries = await fs.readdir(fullSrc, { withFileTypes: true });
  for (const entry of entries) {
    const rel = prefix ? path.join(prefix, entry.name) : entry.name;
    const srcPath = path.join(cwd, rel);
    const dstPath = path.join(target, rel);
    if (bindSet.has(rel)) {
      await fs.symlink(srcPath, dstPath);
      continue;
    }
    if (entry.isDirectory()) {
      await fs.mkdir(dstPath, { recursive: true });
      await walkAndMaterialize(cwd, target, rel, bindSet, copyOne);
    } else if (entry.isFile()) {
      await copyOne(srcPath, dstPath);
    }
  }
}

    } catch (err) {
      if ((err as NodeJS.ErrnoException).code === 'ENOENT') return;
      throw err;
    }
    for (const entry of entries) {
      const rel = prefix ? path.join(prefix, entry.name) : entry.name;
      if (entry.isDirectory()) {
        stack.push(rel);
        continue;
      }
      const overlayPath = path.join(overlayRoot, rel);
      const targetPath = path.join(target, rel);
      await fs.mkdir(path.dirname(targetPath), { recursive: true });
      await fs.copyFile(overlayPath, targetPath);
    }
  }
}

async function materializeFile(src: string, dst: string, mode: TxCopyMode): Promise<CopyResult> {
  if (mode === 'copy') {
    await fs.copyFile(src, dst);
    return 'copied';
  }
  // reflink path
  try {
    if (process.platform === 'darwin') {
      await execFileAsync('cp', ['-c', src, dst]);
    } else {
      await execFileAsync('cp', ['--reflink=always', src, dst]);
    }
    return 'reflinked';
  } catch (err) {
    if (mode === 'reflink') throw err;
    await fs.copyFile(src, dst);
    return 'copied';
  }
}
```

```ts
export const diffSanity: Validator = {
  name: 'builtin:diff-sanity',
  defaultSeverity: 'blocking',

    const start = Date.now();
    const findings = [];
    const diffPath = diffJsonPath(path.join(resolveCliqHome(), 'tx'), ctx.txId);
```

```ts
export async function runStageA(ctx: ApplyContext): Promise<StageAOutcome> {
  return withTxLock(ctx.root, ctx.txId, async () => {
    // A1a
    const tx = await readTxState(ctx.root, ctx.txId);
    if (!tx) throw new Error(`tx not found: ${ctx.txId}`);
    if (tx.state !== 'approved') throw new ApplyRejected(`tx state is ${tx.state}; cannot apply`);
    if (await readAbortProgress(ctx.root, ctx.txId)) throw new ApplyRejected('tx is being aborted; cannot apply');
    if (await readApplyProgress(ctx.root, ctx.txId)) throw new ApplyRejected('apply already in flight; use cliq tx status or wait');
    // A2 index-clean recheck (call validators/builtin/index-clean.checkIndexUnchanged)
    await checkIndexUnchanged(tx.workspaceRealPath, ctx.txId);
    // A3 preflight
    const diff = await readDiff(ctx.root, ctx.txId);
    const plan: { path: string; fingerprint: string }[] = [];
    for (const entry of diff.files) {
      if (entry.op !== 'modify') throw new Error(`unsupported diff op in v0.8: ${entry.op}`);
      const real = await fs.readFile(path.join(tx.workspaceRealPath, entry.path), 'utf8');
      if (real !== entry.oldContent) throw new ApplyConflict(`external change detected at ${entry.path}`);
      plan.push({ path: entry.path, fingerprint: createHash('sha256').update(real).digest('hex') });
    }
    // A4 ghost snapshot
    const ghostSnapshotId = await createApplyPreSnapshot(tx.workspaceRealPath);
    // A5 write apply-progress
    await writeApplyProgress(ctx.root, ctx.txId, {
      phase: 'apply-pending',
      ghostSnapshotId,
      startedAt: new Date().toISOString(),
      filesPlanned: plan.map((p) => p.path),
      filesWritten: []
    });
    return { plan, ghostSnapshotId };
    // A6 lock auto-released by withTxLock
  });
}

export class ApplyRejected extends Error {}
export class ApplyConflict extends Error {}
```

```ts
export async function runStageB(ctx: ApplyContext, plan: PreflightPlan): Promise<StageBOutcome> {
  return withTxLock(ctx.root, ctx.txId, async () => {
    const tx = await readTxState(ctx.root, ctx.txId);
    if (!tx || tx.state !== 'approved') {
      // B1a defense
      await deleteApplyProgress(ctx.root, ctx.txId);
      throw new Error('tx state changed during apply; aborting Stage B');
    }
    let progress = await readApplyProgress(ctx.root, ctx.txId);
    if (!progress) throw new Error('apply-progress missing at Stage B');
    progress = { ...progress, phase: 'apply-writing' };
    await writeApplyProgress(ctx.root, ctx.txId, progress);
    for (const planned of plan) {
      // B3a re-verify fingerprint
      const real = await fs.readFile(path.join(tx.workspaceRealPath, planned.path), 'utf8');
      const fp = createHash('sha256').update(real).digest('hex');
      if (fp !== planned.fingerprint) {
        // partial: transition tx state
        await writeTxState(ctx.root, { ...tx, state: 'applied-partial' });
        await writeApplyProgress(ctx.root, ctx.txId, { ...progress, phase: 'apply-failed-partial', error: { stage: 'B3a', path: planned.path, message: 'fingerprint mismatch' } });
        throw new ApplyPartial(`mid-stage external change at ${planned.path}`, progress.ghostSnapshotId);
      }
      // B3b-d write+fsync+rename
      const target = path.join(tx.workspaceRealPath, planned.path);
      const tmp = `${target}.cliq-tx-tmp`;
      await fs.writeFile(tmp, planned.newContent, 'utf8');
      const fh = await fs.open(tmp, 'r+'); await fh.sync(); await fh.close();
      await fs.rename(tmp, target);
      progress = { ...progress, filesWritten: [...progress.filesWritten, planned.path] };
      await writeApplyProgress(ctx.root, ctx.txId, progress);
    }
    progress = { ...progress, phase: 'apply-committed' };
    await writeApplyProgress(ctx.root, ctx.txId, progress);
    return { ghostSnapshotId: progress.ghostSnapshotId };
  });
}

export class ApplyPartial extends Error {
  constructor(message: string, public ghostSnapshotId: string) { super(message); }
}
```

### Task 28: Stage C — session-first then tx transitions

Stage C is split into two sequential phases to match the spec's required ordering: session fsync MUST happen before tx state.json transitions. The existing `mutateSession` helper saves session AFTER the callback returns; doing tx writes inside the callback would invert the order and create an exactly-once gap (a crash between in-callback tx writes and post-callback session save would leave tx=`applied` with no `tx-applied` record, and recovery would skip the tx because `apply-progress.phase=apply-finalized` is terminal).

**Phase C-session:** `mutateSession` callback does only session-side work (record scan-or-append + conditional `activeTxId` clear). When it returns, session.json has been fsynced.

**Phase C-tx:** Acquire `withTxLock` separately, write `apply-progress.phase=apply-finalized`, then write `tx state.json=applied`.

Crash analysis (each window is recoverable):

- Crash before Phase C-session save: tx state still `approved`, `apply-progress.phase=apply-committed`, no record in session. Recovery (16.4.1 `apply-committed`) re-runs Stage C; C-session idempotency check (record already absent → append; activeTxId still set → clear). Converges.
- Crash after Phase C-session save, before Phase C-tx: session has the record + activeTxId cleared, tx state still `approved`, apply-progress still `apply-committed`. Recovery sees `apply-committed`, re-runs Stage C; C-session idempotency: record present + activeTxId already different → skip both. Phase C-tx: writes apply-progress=apply-finalized then state=applied. Converges.
- Crash inside Phase C-tx between apply-progress write and state write: progress=`apply-finalized`, state=`approved`. Recovery (16.4.1 `apply-finalized` rule) transitions state to `applied`. Converges.
- Crash after both Phase C-tx writes: nothing left to do.

The cross-phase window holds neither lock, but the deterministic record id makes Phase C-session idempotent and the apply-progress phase machine makes Phase C-tx idempotent. The reviewer-flagged exactly-once break does not occur.

```ts
// src/workspace/transactions/apply.ts
import { mutateSession } from '../../session/store.js';
import { applyRecordId, validatorSummaryFromTx } from './types.js';
import { withTxLock, readApplyProgress, writeApplyProgress, readTxState, writeTxState } from './store.js';

export async function runStageC(ctx: ApplyContext, ghostSnapshotId: string): Promise<void> {
  // Phase C-session: append record + clear activeTxId; mutateSession fsyncs on return.
  await mutateSession(ctx.cwd, async (session) => {
    const progress = await readApplyProgress(ctx.root, ctx.txId);
    const tx = await readTxState(ctx.root, ctx.txId);
    if (!progress || !tx) return; // nothing to do
    if (progress.phase === 'apply-finalized' && tx.state === 'applied' && session.activeTxId !== ctx.txId) {
      // C3 idempotency no-op: everything already done
      return;
    }
    const recordId = applyRecordId(ctx.txId);
    const present = session.records.some((r) => r.id === recordId);
    if (!present) {
      session.records.push({
        id: recordId,
        ts: new Date().toISOString(),
        kind: 'tx-applied',
        role: 'user',
        content: `Transaction ${ctx.txId} applied: ${tx.diffSummary?.filesChanged ?? 0} files changed`,
        meta: {
          txId: ctx.txId,
          txKind: 'edit',
          diffSummary: tx.diffSummary!,
          files: {
            creates: tx.diffSummary?.creates ?? [],
            modifies: tx.diffSummary?.modifies ?? [],
            deletes: tx.diffSummary?.deletes ?? []
          },
          validators: validatorSummaryFromTx(tx),
          overrides: tx.overridesApplied ?? [],
          artifactRef: `tx/${ctx.txId}/`,
          ghostSnapshotId
        }
      } as SessionRecord);
    }
    if (session.activeTxId === ctx.txId) session.activeTxId = undefined;
    // mutateSession will fsync session on return — session-side terminal write completes here
  });

  // Phase C-tx: under tx-store lock, transition apply-progress then state.
  // Crash between phases is safe: session is already correct; recovery's apply-committed rule
  // re-enters Stage C and the C-session phase becomes a no-op via the deterministic-id check above.
  await withTxLock(ctx.root, ctx.txId, async () => {
    const progress = await readApplyProgress(ctx.root, ctx.txId);
    const tx = await readTxState(ctx.root, ctx.txId);
    if (!progress || !tx) return;
    if (progress.phase !== 'apply-finalized') {
      await writeApplyProgress(ctx.root, ctx.txId, { ...progress, phase: 'apply-finalized' });
    }
    if (tx.state !== 'applied') {
      await writeTxState(ctx.root, { ...tx, state: 'applied', ghostSnapshotId });
    }
  });
}
```

### Task 40: Auto-compact range selector treats tx boundaries as non-splittable

**Files:**
- Modify: `src/session/auto-compaction.ts`
- Modify: `src/session/auto-compaction.test.ts`

Existing range selector respects turn boundaries. Extend so that an explicit multi-turn tx — bounded by a `tx-opened` record at the start and a matching `tx-applied` or `tx-aborted` record at the end (paired by `meta.txId`) — is treated as one non-splittable unit. Cut points that would fall between an open and its matching terminal must move before the `tx-opened` (preferred) or after the terminal record. Implicit per-turn tx do not write `tx-opened` records and are already protected by the existing turn-boundary rule.

The boundary source is concrete and derivable from session records alone — no inspection of `Session.activeTxId` (which is a current value, not a history) and no reference to tx artifacts. Auto-compact remains a session-only consumer per the Section 15 contract.

Algorithm sketch:

```ts
function findExplicitTxSpans(records: SessionRecord[]): Array<{ openIndex: number; closeIndex: number }> {
  const spans: Array<{ openIndex: number; closeIndex: number }> = [];
  const openByTxId = new Map<string, number>();
  for (let i = 0; i < records.length; i++) {
    const r = records[i];
    if (r.kind === 'tx-opened') {
      openByTxId.set(r.meta.txId, i);
    } else if (r.kind === 'tx-applied' || r.kind === 'tx-aborted') {
      const open = openByTxId.get(r.meta.txId);
      if (open !== undefined) {
        spans.push({ openIndex: open, closeIndex: i });
        openByTxId.delete(r.meta.txId);
      }
    }
  }
  return spans;
}

function adjustCutForTxSpans(rawCutIndex: number, spans: Array<{ openIndex: number; closeIndex: number }>): number {
  for (const span of spans) {
    if (rawCutIndex > span.openIndex && rawCutIndex <= span.closeIndex) {
      // Cut would split the explicit tx; move before the open.
      return span.openIndex;
    }
  }
  return rawCutIndex;
}
```

A still-open explicit tx (no matching terminal record yet) means the user has run `cliq tx open` but not `apply`/`abort`. Auto-compact should not cut into that span either — treat unmatched `tx-opened` as forming an implicit span from `openIndex` to `records.length`. Range selector falls back to a cut before `openIndex` if reducing further is required.
